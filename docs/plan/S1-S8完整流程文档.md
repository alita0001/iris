# IRIS S1–S8 流程文档：实现、资产与阻塞项

> 状态日期：2026-07-13。
> 本文严格区分“代码接口存在”“有 legacy/pilot 资产”“有正式 point-grounded 资产”。当前正式 point-grounded 资产为 **0**。

## 0. 数据纪律

1. recovery 结论必须来自 execute–then–undo trace；LLM 只能提案动作或改写措辞。
2. `RECOVERED` 是构造性、solver/预算/σ 相对的证据。
3. `NOT_RECOVERED_WITHIN_BUDGET` 是有限搜索失败，不是不可逆证明。
4. recoverability 与 normative risk 独立；恢复状态不等于撤回现实伤害。
5. 无 `probe_point_id/state_id/action_instance_id` 的记录不得进入正式训练。
6. 不覆盖原始 JSONL；legacy、mock、失败轨迹和开发数据必须显式隔离。

## 3. S1：日志、指纹、历史与 AXTree

### 3.1 不可变 collection run

新 collection attempt 使用不可变 `run_id` 和 physical trajectory identity。重复 logical task/seed 是独立 run，不能覆盖 raw 或假装成同一独立样本。旧 meta 中 93 行只有 31 个 logical trajectory IDs；论文统计不得把 93 当作 93 条独立轨迹。

### 3.2 统一 train/deploy 输入

唯一 serializer：`revact/prompts.py::build_policy_messages`。

```text
system
user:
  <goal>...</goal>
  <history>last K observed-delta rows</history>
  <observation>current pruned AXTree</observation>
```

- `POLICY_HISTORY_STEPS=6` 同时驱动 collection、IrisPolicy、rollout 和新 materializer；
- multiturn history flag 由相邻观测 diff 重算，不使用 grounded label；single reached-state 路径目前只信任 `history_source=trajectory` 声明，尚未与 raw run/hash 绑定；
- `no-effect` 表示可见状态未变化，不代表动作安全；
- action-anchored pruning 保留 supervised bid 和祖先上下文；
- 缺失 supervised bid 时 assembler/trainer 直接拒绝。

旧 multi SFT 中 risky bid 仅 12/62 可见；新输入 gate 不允许直接训练该文件。

## 4. S2：轨迹与关键状态发现

`afforded_action_types` 仍只覆盖 `add_to_cart/place_order/delete_address`，但新 collector 已把它降为 `keyword_ranking_hints`，不再作为发现门；候选状态来自全部 legal interactive controls。legacy state bank 241 行的 210/29/2 分布仍是旧 shallow-keyword 产物，反映 Magento 页面文字和 collector 路径，不是 web action 的自然分布。

代码中的 mutation detector 协议为：

1. reset 到可恢复 snapshot；
2. 枚举全部合法 interactive controls；
3. 安全执行候选；
4. 比较持久 UI/API/DB signal；
5. reset；
6. 保存 mutation candidate 和 detector provenance。

关键词、LLM 和 sitemap 只能排序/覆盖采样，不得产生 label。当前 240-control 数字仅来自离线 fixture；正式上线前仍需在至少 200 个 live controls 上报告 mutation precision/recall 与 Wilson interval。

## 5. S3：状态到达

`reach.py/scale.py` 的稳定 URL 和运行时 bid 解析可用于创建候选状态。正式 point 还要求：

- state/action identity 不由 action-type 关键词推断；
- pre/action/post observation 和 signal hash 完整；
- run、账户、权限、seed 可复放；
- target bid 在未截断当前输入中可见。

历史 `canonical_history` 和语义 plan 仅是开发兼容，不允许进入正式主训练集。

## 6. S4：候选动作

S4 不能继续缺号。它同时服务：

- S5：给 probe 提供待测 `(state, action)`；
- S8：生成与部署分布一致的 DPO negatives；
- eval：构造非专家动作和 decoy challenge。

正式 candidate 至少记录：

- `candidate_id/state_id/snapshot_hash`
- canonical action、raw action、bid
- `candidate_category/source/legal_at_snapshot`
- proposer model/version（若由模型提案）

每状态目标 4–6 个候选：专家、普通控件、安全替代、constraint/goal violation、decoy、VERIFY、on-policy error。LLM 不得填写 recovery label。

formal consumer 使用 `data/raw/candidates/iris_candidates.v3.jsonl`：300 条 `iris.candidate.v2`、50 个 state，其中 250 条来自当前 AXTree 枚举、50 条为 producer-declared `source=expert` target bid，均通过 bid/snapshot-hash gate。后者没有 trajectory/run/collector-success provenance，不是成功专家示范的证据。当前 v3 **没有** decoy 或 on-policy row；旧 v1/v2 文件只作开发资产，不进入 formal DPO。候选尚未与 point label 或 DPO pair 关联。历史 DPO 按生成机制是 synthetic flip，但 385/385 行缺 `negative_source`，只能审计为 `legacy_unspecified`；formal validator 要求 legal/on-policy negatives 比例至少 50%，因此 formal DPO dry-run 仍应失败。

## 7. S5：point-level grounding

### 7.1 Canonical ontology

```text
effect_status:
  CHANGED | NO_EFFECT | UNKNOWN

recovery_status:
  RECOVERED | PARTIALLY_RECOVERED |
  NOT_RECOVERED_WITHIN_BUDGET | UNKNOWN

undo_cost_steps: nullable integer
```

新 formal body 永不写 `IRREVERSIBLE`。legacy reader 可映射该词，但不得提升证据强度。

### 7.2 Point admission

`GroundingPoint.validate(formal=True)` 要求完整 point identity、transition hashes、signals、solver set、budget 和 evidence。`save_probe_points` 同时维护 body/manifest 1:1 和 immutable ID。

普通 `save_results` 只写 `grounded/smoke/`，不会再追加冻结的 legacy body。`revact probe --formal-spec ...` 可提供 label-free point identity 并调用 `save_formal_probe_results`，但当前注册探针尚未填齐 pre/post/final signals、observation hashes 与完整 solver traces，因此会被 formal schema 拒绝，而不会回退到 class label。正式 live producer 仍是阻塞项。

### 7.3 Negative protocol

正式负点需要 deterministic controller + affordance BFS + LLM undo attacker，任一恢复即翻标签，并报告 attack break rate。当前主要只有固定站点 controller；在多 solver 进入前，不得报“不可逆率”。

### 7.4 σ 的边界

当前 signals 主要是结构化 UI 读数，不是数据库上界。它可能漏掉邮件、支付、webhook、审计记录、缓存、异步状态和第三方观察。每个 action class 都必须发布 signal coverage 声明。

## 8. S6：decision truth

正式 truth 由独立轴组成：

- `effect_status/recovery_status`
- `normative_risk` 或 `policy_constraint_truth`
- `action_required_for_goal`
- `violates_constraint`
- `expected_decision`

正式 oracle 不从 recoverability 单独推出安全结论。历史固定 risk 数值只能用于 legacy compatibility，不能报 ECE。

## 9. S7：teacher 与 prompt provenance

正式 teacher 只能改写由 point evidence pin 住的事实。QC 检查：

- 标签结构和顺序；
- input observation evidence；
- measured post diff；
- undo action/cost；
- recovery、decision 和 answer consistency；
- `prediction_source=probe_transition`；
- `undo_source=probe_point_id`。

prompt 以 content-addressed bundle 保存全文、fingerprint 和父版本；另用 `prompt_generation_fp` 对 producer/model/decode 配置独立寻址。single/multiturn assembler 和 teacher 路径已写入该 provenance，formal export 会携带两类 bundle。历史 `prompts_fp=259fceac4e0c` 仍缺全文 bundle，且当前 distilled 产物仍为 0；代码存在不等于 teacher 已生效。

工作台已能读取、浏览并导出 single/multiturn SFT/DPO，也展示候选和 legacy grounding。但现有样本仍是历史资产；在 formal point 为 0 时，`state → candidate → transition → probe → label → teacher → split` 的正式链条不可能完整。

## 10. S8：assembly、split 与训练

### 10.1 Formal assembly

正式样本必须通过唯一 `probe_point_id` join，且：

- input 使用共享 stateless builder；
- risky bid 可见；
- prediction/undo 引用 transition/point evidence；
- `is_mock=false`；
- expert trajectory 必须声明 `collector_success=true`，并在非空 formal 集中与不可变 collection run 精确连接；当前 single 代码仅检查布尔字段；
- assistant turn type 显式；
- tokenizer 超限 count=0。

当前 formal points=0，所以正确的正式组装结果是无可训练样本，而不是回退 action-type label。

### 10.2 Split

`data/splits.py` 对 single 与 multiturn 做一次联合 connected-component 分配，以 state/entity/template/environment 为 publication isolation 轴，并额外报告 action/site/privilege overlap。train/dev/test 三侧必须非空；任何跨 family 重叠、旧 shard 残留或缺失 `SPLIT_REPORT` 都使 formal export fail closed。cross-action、cross-privilege、cross-template、cross-site 只有在至少两个非空组且 train/test 都非空时才标 `available=true`。

代码能生成 challenge split 不代表实验已经存在。`*_cross_<axis>*.jsonl` 路径可被
物化为 0 行 stale-clearing sentinel；只有 `SPLIT_REPORT` 标记 `available=true` 且两侧
非空才是可用 split。当前历史 single 文件只有 shopping，cross-site 结果不可用。

### 10.3 训练定位

- SFT：completion-only，formal validator fail closed；
- DPO：synthetic 和 deployment negatives 分层；当前 0% deployment negatives，formal gate失败；
- GRPO/GSPO：固定 prompt 的 offline RLVR ablation，不是环境在回路的多轮 RL。

没有正式数据前，不启动 GPU 训练。

## 11. 评测

`eval/metrics.py` 实现指标定义和 numerator/denominator provenance；`eval/rollout.py` 接入新口径。代码测试通过不等于有正式模型结果。

旧 `FSR=9.1%` 已撤回。旧唯一计数失败是可恢复 add-to-cart，因此不属于 true-danger FSR。旧日志只保留为 harness 和 failure-case 调试资产。

任何 `n<30` cell 只列 count/interval。正式报告需加 0/5/10/20% label-noise sensitivity 和 UNKNOWN bounds。

## 12. 安全 Runbook

只读/离线：

```bash
git status --short
pytest -q
python scripts/migrate_grounding_v3.py --help
python -m revact.cli train --dry-run
python -m revact.cli train-dpo --dry-run
python -m revact.cli train-grpo --dry-run
```

注意：当前 formal 数据为空，训练 dry-run 应报告阻塞，而不是继续训练 legacy 资产。

live smoke probe：

```bash
source scripts/export_webarena_env.sh
conda run -n agentlab python -m revact.cli probe --list
conda run -n agentlab python -m revact.cli probe --all-nondestructive
```

上述命令写独立的 `grounded/smoke/` artifact，不修改冻结 legacy 文件，也不是 formal point producer。只有带完整 label-free spec 且探针确实产出完整 transition evidence 的 `--formal-spec` 路径才能进入 point schema；当前注册探针尚未达到该条件。

破坏性动作必须同时具有项目负责人批准、`--commit` 和 `REVACT_ALLOW_DESTRUCTIVE=1`。禁止在训练 rollout 中真实下单、发布、删除或支付。
