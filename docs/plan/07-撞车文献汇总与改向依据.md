# 撞车文献汇总与改向依据

> 本文回答三个问题：每篇相关工作**做了什么**（数字已联网核验，2026-07-02）；它与我们最初的
> [`WCAM整体技术方案.md`](../WCAM整体技术方案.md)（下称 WCAM）**哪一条主张直接撞车**；以及
> **为什么我们放弃该方向、改向 IRIS**（[`06-IRIS项目计划书.md`](06-IRIS项目计划书.md)）。
> 结尾给"WCAM 原始主张 → 被谁占了 → IRIS 怎么改"总表。
>
> 核验说明：标 ⚠️ 的条目是我们早期文档里的**事实错误**，已在本文纠正。

## 一、世界模型主线上的四篇

### 1. VAGEN — Reinforcing World Model Reasoning for Multi-Turn VLM Agents（arXiv:2510.16907, NeurIPS 2025）

**① 做了什么**。把多轮 VLM agent 建模为 POMDP，令模型在 `<think>` 内显式输出 `<observation>`（状态估计）与 `<prediction>`（转移建模），再输出 `<answer>`。训练用 **WorldModeling Reward**（以环境真实反馈/reference state 密集监督中间预测）+ **Bi-Level GAE**（turn 级优势下传到 token 级）+ PPO。3B 模型在 5 个视觉/具身 benchmark（导航、Sokoban、机械臂等）平均 0.82，约为未训练版（0.21）的 3.9 倍，超过 GPT-5（0.75）、Gemini 2.5 Pro（0.67）、Claude 4.5（0.62）。

**② 与 WCAM 哪里撞**。WCAM §6"模型架构设计"原文承认"一条训练序列应尽量对齐 VAGEN 的 world-modeling 格式"——`<observation>/<reasoning>/<prediction>/<answer>` 四段式、POMDP 表述、"把世界模型推理写进自回归输出并用环境反馈监督中间预测"这三件事 WCAM 全部依赖，而它们正是 VAGEN 的三个署名贡献。WCAM §12"预期贡献"第一条"Consequence-Centric World-Consequence-Action Modeling"在审稿人眼里就是"VAGEN 格式 + 几个附加标签"。

**③ 为什么不做**。"内在化世界模型推理"是综述 2606.09032 §4.1 有专门标题的既有范式（VAGEN 归 §4.1.2 "world model in the reasoning trace"）。在已命名范式内做增量标签设计，novelty 判定必然是"标签工程"。

**④ IRIS 的 delta**。格式与内在化明写承自 VAGEN；IRIS 改变的是**世界模型预测的对象**（前向转移 → 转移的可逆性结构 + undo 构造）与**监督来源**（reference next-state → execute-then-undo 行为测量），并把"内在化"降格为受控实验的自变量。

### 2. Qwen-AgentWorld — Language World Models for General Agents（arXiv:2606.24597, 2026）

**① 做了什么**。首个大规模通用语言世界模型：CPT→SFT→RL 三阶段，**10M+ 环境交互轨迹**，覆盖 MCP/Search/Terminal/SWE/Android/Web/OS 七类环境；发布 **35B-A3B 与 397B-A17B** 两个 MoE 模型；两种用法——decoupled 环境模拟器（供 agentic RL/搜索，胜过纯真实环境训练）与 agent foundation model（世界模型预训练作为下游 agent 的 warm-up，7 个 benchmark 提升）；AgentWorldBench 上超过 GPT-5.4。

**② 与 WCAM 哪里撞**。WCAM §8"训练策略"与 §2.5 里"让模型学会预测状态变化"的部分——凡是往"训练 web 世界模型/next-state prediction"上靠的表述，都会被这个 10M 轨迹、397B 参数的工程直接盖掉。WCAM 自己在 §2.5 已经列了这条红线，但正文多处（如 §7 数据构建"记录 O_t 到 O_{t+1} 的变化"作为核心资产）仍在暗示转移预测是贡献的一部分。

**③ 为什么不做**。资源上不可竞争（我们无 10M 轨迹、无 35B+ 算力），方向上没有意义——它是基础设施，不是研究问题。

**④ IRIS 的 delta**。不训模拟器、不比拟真度；Qwen-AgentWorld 回答"动作后环境返回什么"，IRIS 回答"返回之后 agent 还能不能自己回来、代价多少"——后者不在其 10M 轨迹的监督信号里。

### 3. WebWorld — A Large-Scale World Model for Web Agent Training（arXiv:2602.14721, 2026）

**① 做了什么**。1M+ 开放网页交互训练的浏览器世界模型（8B–32B），模拟视野 30+ 步；WebWorld-Bench 上比肩 Gemini-3-Pro；用其合成轨迹训练 Qwen3-14B 在 WebArena +9.2%；作为推理时搜索的世界模型胜过 GPT-5。

**② 与 WCAM 哪里撞**。与 Qwen-AgentWorld 同一条：WCAM 中一切"web 环境 next-state 预测"相关的表述。WebWorld 把这条线在 **web 垂直域**也占满了。

**③ 为什么不做**。同上；且 WebWorld 只做前向模拟、不建模后果/可逆性/安全约束——它的存在恰好证明前向线饱和而逆向线空白。

**④ IRIS 的 delta**。WebWorld 是 IRIS 实验 0（H1 独立性探针）的理想被试：拿前向训练的模型测可逆性预测，若它测不出来，"可逆性是前向学不到的独立能力"即获得实证。

### 4. 综述 — Bridging the Agent-World Gap: Text World Models for LLM-based Agents（arXiv:2606.09032, 2026）

**① 做了什么**。系统综述 text/language world models：§4.1 "Internalizing World Models into Agent Parameters"（§4.1.2 "world model in the reasoning trace" = VAGEN 类）、§5.2 "World Model as Verifier"（= WMA/WAC 类）、§7 开放问题含 §7.1 "World Model–Policy Coupling" 与 §7.4 "Grounding, Adaptation, and Generalization"。全篇几乎不谈 safety/reversibility。（章节标题已逐条核验属实。）

**② 与 WCAM 哪里撞**。WCAM §4 核心思想"将后果预测嵌入自回归输出序列……避免多模块误差传递"——这段话就是 §4.1 范式的复述。把它当贡献写，等于宣称发明了一个综述已经给了抽屉编号的东西。

**③ 为什么不做**。范式命名权已丢失；但综述同时送了两件礼物：§7.1 把"内在 vs 外置该耦合到什么程度"点名为开放问题，且全篇 safety/reversibility 空缺。

**④ IRIS 的 delta**。IRIS 的 coupling 受控研究就是对 §7.1 的直接作答；grounded reversibility 落在综述完全空着的缝里。引用综述反而成为定位利器而非威胁。

## 二、安全/风险线上的五篇（WCAM 当年完全没引的真正杀手）

### 5. WMA — Web Agents with World Models（arXiv:2410.13232, ICLR 2025）

**① 做了什么**。给冻结的 LLM 策略外挂世界模型做动作选择；关键设计 transition-focused observation abstraction（用自然语言只描述相邻步的重要状态差异，回避完整 HTML 预测）；WebArena/Mind2Web 上提升策略选择且比树搜索省钱。**其 motivation 原文即"agent 重复购买不可退机票"——不可逆错误。**

**② 与 WCAM 哪里撞**。WCAM §1"研究动机"的核心叙事（"缺乏后果意识→不可逆错误→执行前想象"）**2024 年 10 月就被 WMA 写进了 ICLR 论文的第一段**。WCAM §2.2 引了 WMA 却低估了它：把"不可逆"当自己 motivation 的新颖处是站不住的。

**③ 为什么不做**。motivation 层面无 delta 可言；WMA 没做的是把不可逆性**测量出来、学进策略**——它只是拿"不可逆"吓唬读者然后去做前向预测。

**④ IRIS 的 delta**。WMA 说"agent 应该怕不可逆"；IRIS 定义、测量并训练"什么是不可逆"。

### 6. WAC — World-Model-Augmented Web Agents with Action Correction（arXiv:2602.15384, 2026）

**① 做了什么**。Action Model 提候选 → World Model 模拟 outcome → Judge 打分 {0, 0.5, 1} + rationale → 低于阈值 θ 触发修正反馈。VisualWebArena 24.5%（vs WebDreamer 22.7%，+1.8%）、Online-Mind2Web +1.3%。**只报 task success；无风险 taxonomy、无可逆性、无安全指标、无用户约束。**

**② 与 WCAM 哪里撞**。WCAM §1 末段与 §2.3 的表述（"执行危险动作造成损失、不可逆轨迹偏离，应在执行前模拟后果并修正"）与 WAC 的 motivation 几乎同义；WCAM §11 只把它列为"Modular baseline"，严重低估——**"web agent 执行前用世界模型模拟后果避险"这整个故事已经是 WAC 的了**。

**③ 为什么不做**。叙事已被占，且 WCAM 的"统一模型 vs 模块化"对比若无对齐监督就只是架构口味之争。

**④ IRIS 的 delta**。WAC 从对手变成**对照腿**：decoupled 臂按 WAC 结构搭建、用与 coupled 完全相同的 grounded 标签训练，架构差异被隔离成唯一自变量——这才是能发表的比较。

### 7. ST-WebAgentBench — Safety & Trustworthiness Benchmark（arXiv:2410.06703, **ICLR 2026**）⚠️

**① 做了什么**。安全=避免非预期/不可逆动作，可信=遵守组织/用户/任务三层 policy；**当前 v5 为 222 个企业场景任务**（⚠️我们早期文档写"ICML'25、375 任务"是错的，投稿引用必须用 ICLR 2026 / 222），6 个安全维度、Completion-under-Policy（CuP）与 Risk Ratio 指标；发现启用 policy 后 SOTA agent 完成率大跌、违规集中在 user-consent 与 strict-execution。

**② 与 WCAM 哪里撞**。WCAM §10"实验设计"第一/二类评测（False-Safe Rate、Unnecessary Confirmation、用户约束违背评测）与 §7 的约束数据构建——**自建安全评测框架**这件事被它用 3,000+ policy 实例级的工程做掉了；WCAM 的 `<NEED_CONFIRMATION>` 对应它的 user-consent 维度。

**③ 为什么不做**。自建 benchmark 会被一句"ST-WebAgentBench 已存在且更大"打回；且它用规则/人工 policy 判定，不解决我们的监督问题。

**④ IRIS 的 delta**。不再造轮子：复用其 shopping 子集作外部效度（CuP/Risk Ratio 直接报），我们的贡献退到它不做的两件事——行为化测量与策略训练。

### 8. WebGuard — Building a Generalizable Guardrail for Web Agents（arXiv:2507.14293, 2025）【早期文档完全漏掉】

**① 做了什么**。首个 web 动作风险预测数据集：**4,939 个 state-changing 动作、193 个网站、SAFE/LOW/HIGH 三级人工标注**；前沿 LLM 预测动作后果准确率 <60%、高危召回不足；微调（含 Qwen2.5VL 系）显著提升，做成外置 guardrail 供人审批。

**② 与 WCAM 哪里撞**。**这是对 WCAM §5"Consequence Token 设计"最致命的一篇**：Risk Level Tokens（`<RISK_NONE/LOW/MEDIUM/HIGH>`）与 Risk Type Tokens 所对应的"动作风险分级预测"已被 WebGuard 用近 5 千条**人工**标注做掉——WCAM 计划用规则+LLM 打这些标签，质量下位替代。WCAM §7 的"风险标签多层流程（规则→LLM→抽检）"同样被覆盖且被超越。

**③ 为什么不做**。用 LLM 意见标签去追人工意见标签的数据集，双输：规模输、可信度输。意见类风险分级这条路必须整条放弃。

**④ IRIS 的 delta**。监督来源换轴：WebGuard 回答"人认为多危险"，IRIS 回答"环境里实测能否撤销、几步、剩多少残差"；`risk_type/risk_level` 从 IRIS 的监督目标中删除（只留分析用），并用"意见 vs 行为分歧矩阵"定量展示意见标注在可逆性上的失效点。

### 9. SafePred — A Predictive Guardrail for Computer-Using Agents via World Models（arXiv:2602.01725, 2026）【早期文档完全漏掉】

**① 做了什么**。用世界模型生成短期/长期风险的语义表征，把"预测的未来风险"对齐到当前决策，step 级干预 + task 级重规划的外置 risk-to-decision loop；LoRA 蒸馏到小模型；报 97.6% 安全率、相对反应式基线任务效用 +21.4%。

**② 与 WCAM 哪里撞**。WCAM §4"保留世界模型的执行前想象能力用于安全"——**"用世界模型做 agent 安全"的第一篇已经出现了**。若 WCAM 原样投稿，Related Work 里绕不开"SafePred 已用世界模型做预测式安全防护"。

**③ 为什么不做**。"首个把世界模型用于安全"的声明死了；SafePred 仍是外置+前向+意见 policy 驱动。

**④ IRIS 的 delta**。SafePred 恰好补齐了 coupling 对照的外置端代表性（WAC 选候选、SafePred 做 guardrail，都是 decoupled）；IRIS 的问题是它们都没问的："这套后果推理放进策略参数里，相对放在外面，买到什么？"——且 IRIS 的风险信号是行为实测而非 policy 文本。

## 三、可逆性构念的四篇（防"我们发明了可逆性"的口径事故）

### 10. Leave No Trace（Eysenbach et al., arXiv:1711.06782, ICLR 2018）

**①** 联合训练 forward policy 与 reset policy，用"能否回到初始状态"做安全 RL 的中止判据，减少不可逆状态进入与人工 reset。**②** 与 WCAM 无直接撞车（WCAM 没引 RL 线），但它证明"可逆性作为安全信号"在 RL 里是 2018 年的想法——WCAM §5 把 Recoverability Tokens 当新构念的写法在懂 RL 的审稿人面前会翻车。**③** 不做纯 RL 可逆性：环境是低维模拟器，与 LLM web agent 的语义动作空间不可比。**④** IRIS 差异：reset 策略是隐式学习的；IRIS 的 undo 是显式构造、逐点执行验证、附带代价与残差的测量协议。

### 11. Grinsztajn et al. — There Is No Turning Back（arXiv:2106.04480, NeurIPS 2021）

**①** 自监督估计可逆性：从随机轨迹学 ψ(s,s′)（s→s′ 相对 s′→s 的经验可达性序），用于 reward shaping（RAE）与动作拒绝（RAC），在 CartPole/Sokoban 等环境实现零不可逆失误。**②** 同上，构念先占。**③** 分布性估计需要海量轨迹且不给出"怎么撤销"；web 环境采不起也不需要。**④** IRIS 差异：构造性（给出 undo 动作序列）、真实站点后端信号判据、权限维度显式化；Related Work 单列 RL 可逆性一节全引。

### 12. Learning to Undo — Rollback-Augmented RL with Reversibility Signals（arXiv:2510.14503, 2025）

**①** 给 RL 加 rollback 原语与经验可逆性度量 Φ(s,a)，在 CliffWalking/Taxi 类环境提高安全探索效率。**②③** 同上：玩具环境、依赖模拟器 rollback。**④** IRIS 的 undo 只用 **agent 自身动作空间**（不开模拟器后门），这正是 agent-relative 定义的意义——部署时没有 rollback 原语可用。

### 13. Revisable by Design — Reversibility Classes for LLM Agent Actions（arXiv:2604.23283, 2026）

**①** 给 LLM agent 动作执行系统提出可逆性代数分类：idempotent / reversible / compensable / irreversible，用于流式执行的提交/撤销语义设计。**②** 与 WCAM §5 Recoverability Tokens 的 taxonomy 撞：分类命名权已被拿走。**③** 它是系统理论文，无测量、无数据、无学习。**④** IRIS 采用兼容口径（PARTIALLY_RECOVERABLE ≈ compensable）并引用之；贡献差在给分类配上**行为测量协议 + 学习 + 决策消费**。

### 14. 评测红海一笔带过：R-Judge（2401.10019）/ OS-Harm / Agent-SafetyBench

LLM agent 风险意识/安全评测已高度饱和，均为静态判断或意见标注。WCAM §10 若自建"风险意识评测"必然沉没。IRIS 不做通用安全评测，只做可逆性这一个构念的测量与学习，评测外包给 ST-WebAgentBench。

## 四、总表：WCAM 原始主张 → 被谁占了 → IRIS 怎么改

| WCAM 原始主张（出处） | 被谁占了 | IRIS 的改法 |
|---|---|---|
| 内生世界模型推理格式 `<observation>/<prediction>/<answer>`（§6） | VAGEN + 综述 §4.1.2 | 保留为载体，Related Work 明写出处；非贡献 |
| "把后果预测嵌入自回归、避免模块误差传递"（§4） | 综述 §4.1 命名范式 | 内在化降格为 §7.1 受控实验的**自变量** |
| next-state / 状态差异预测（§7 数据核心） | Qwen-AgentWorld、WebWorld | `<prediction>` 只作中间监督；预测**对象**换成 invertibility |
| "执行前模拟后果避险" motivation（§1, §2.3） | WMA（2024-10 已写）、WAC、SafePred | motivation 重写为"前向预测≠知道能否撤销" |
| Risk Type / Risk Level Tokens（§5） | **WebGuard**（4,939 人工标注） | **从监督中删除**，只留分析；用行为标签替代意见标签 |
| Recoverability Tokens / 可逆性 taxonomy（§5） | RL 四篇 + Revisable by Design | 构念收窄为 agent-relative **operational** reversibility + 行为测量协议（这是唯一没人做的） |
| Decision Tokens + NEED_CONFIRMATION（§5） | ST-WebAgentBench user-consent 维度 | 保留，但对齐 ST 口径 + 连续 risk 分数做校准 |
| 自建安全评测/False-Safe 指标体系（§10） | ST-WebAgentBench（ICLR 2026, 222 任务） | 不自建；复用其 shopping 子集 + 在自有 grounded 探针上报 FSR/FBR/校准 |
| 反事实高危动作数据集（§7, §12） | WebGuard（占数据）、WAC（占用法） | 反事实只服务 DPO 对与分歧矩阵；核心资产改为**逐点 grounded 探测记录（含 undo 代价与 MANIFEST 溯源）** |
| "统一模型 vs 模块化更高效"（§6, §13） | WAC/SafePred 单臂存在，无对照 | 升级为**监督对齐的 coupled vs decoupled 受控研究**（成本/校准/泛化三维实测，允许被证伪） |

**一句话结论**：WCAM 的十条主张里，八条的地盘在 2024-10 至 2026-06 之间被逐一占掉；幸存的两条——**行为实测的可逆性**与**监督对齐的耦合对照**——恰好互相成就（前者给后者公平的共同底座），这就是 IRIS 的全部，也是它够一篇论文的原因。
